# HPM-DT 长期路线图

## 路线图原则

HPM-DT 的版本演进以平台能力为主线，不以单个论文功能为主线。每个阶段都必须产出可运行软件、可复现实验、可审计报告和可继续扩展的工程接口。

V2.0B 以后，路线图从 Verification First 调整为 Scene First。默认首页必须是场景工作台，验证中心必须位于工具入口；新增功能优先回答“用户能分析什么新场景/任务”，而不是只增加指标页、评分页或后台审计页。

## V2.x：科研 CAE 平台成型阶段

### V2.0A：可信度验证体系

状态：已实现。

目标是建立 Verification & Validation 层，使平台能自动回答公式实现是否正确、理论算例能否复现、不同后端在哪些场景下可信，以及不确定度和敏感性如何量化。

当前交付包括：

- 解析阵列因子、扫描波束、Green 函数验证；
- MUSIC/ESPRIT/PAWR 测向基准；
- MVDR/LCMV 约束响应验证；
- 传播后端退化审计；
- Monte Carlo 不确定度、OAT 敏感性和可信度评分；
- 中文 HTML 报告、LaTeX 表格、论文图包和 Notebook。

### V2.0B：三维场景编辑器（Scene First）

状态：进行中，已接入 Three.js 三维工作台原型，并已把 Web 首页从验证中心切换为场景编辑。

目标是把 Workbench Layer 从表单式参数配置推进到可视化工程建模：

- 打开软件即进入三维场景；
- Three.js 或 VTK 交互视口；
- 阵列、目标、保护区、反射面、孔缝、腔体拖拽建模；
- 材料、坐标系、观察面、求解区域可视化；
- 工程对象树、属性面板、撤销重做和工程快照；
- 求解、结果档案、可信度复核和论文导出围绕当前场景展开。

当前原型已经完成：

- 默认首页为“场景编辑”，首屏展示 `场景 / 对象 / 任务 / 结果`，左侧主链路为项目管理、场景编辑、阵列设计、传播环境、求解设置、结果查看和论文导出；可信度验证位于工具入口；
- 本地 Three.js 视口；
- 工程对象树和属性面板；
- 阵列、观察面、目标区、保护区、反射面、孔缝、腔体、远场源和材料库的场景 JSON；
- 目标区、保护区、反射面、孔缝、腔体和材料代理的受控属性更新 API；
- 材料设置面板与材料代理审计，支持反射幅度、相位、损耗、粗糙度等归一化代理参数编辑，并把字段白名单、参数范围、引用完整性和安全边界纳入 `MAT-AUDIT-001`；
- 目标区、保护区、孔缝和腔体的三维拖拽移动；
- 属性面板受控几何变换控件，支持对目标区、保护区、孔缝和腔体等可编辑对象进行平移、缩放与旋转微调，并回到后端工程模型校验；
- 当前三维工程状态的快速求解联动，输出指标摘要、对象级约束、适用性提示和 V&V 诊断区；
- 求解任务队列可生成 `JOB-xxxx` 任务记录，绑定 `SOL-xxxx` 结果档案、scene_hash、field_hash 和关键摘要，并维护轻量索引与启动恢复；
- 求解任务生命周期控制支持本地后台 worker 预览、提交时 `CAEProject` 快照、暂停/恢复检查点、重试派生、取消请求、操作日志和 `audit.json`/`audit.csv` 审计报告；当前后台 worker 仍是单机本地线程预览，后续门槛是正式多用户调度器和工程/结果数据库管理；
- 观察面归一化场强结果图层回写，三维视口可叠加半透明 heatmap，求解面板可读出色标、峰值坐标、中心剖面采样和 x/y 剖面交互曲线；
- V&V 适用性 payload 可在求解面板中展开为得分、结论、状态计数和需关注检查项；
- 目标区/保护区对象级指标列表与三维状态徽标，可从结果诊断直接定位到工程对象；
- 求解结果档案列表、详情复查、单次结果 JSON 导出，以及 `index.json`/`index.csv` 轻量结果索引与启动恢复；
- 统一工程资产台账，把 `JOB-xxxx` 求解任务、`SOL-xxxx` 求解结果和 `SNP-xxxx` 工程快照汇总为 JSON/CSV/SQLite 索引，提供类型/关键字筛选、审计报告、资产谱系追溯和前端资产入口；
- SQLite 资产数据库增加 `workbench3d_solve_job_events` 事件表，把 JOB 操作日志展开为可查询数据库行，并提供数据库审计 API 和前端入口；
- SQLite 数据库进一步增加 `workbench3d_solve_jobs`、`workbench3d_results`、`workbench3d_snapshots` 和 `workbench3d_database_manifest` 表，并提供数据库记录浏览 API，形成正式工程/结果数据库迁移前的可审计读模型；
- 资产谱系进一步输出 `lineage.json` 与 `lineage.csv`，把快照、任务、结果、重试和事件派生关系显式化，为正式工程数据库的结果血缘与可重复性审计打基础；
- 资产/快照命名规范审计固化 `JOB-xxxx`、`SOL-xxxx`、`SNP-xxxx` 编号、`scene_hash`/`field_hash` 文件名绑定和 `naming_audit.json`/`naming_audit.csv` 证据，为后续正式工程数据库迁移提供稳定键；
- 撤销/重做历史栈；
- 工程快照创建、落盘、启动恢复、轻量索引和差异对比；
- 后端 `CAEProject` 几何/材料校验和 `scene_hash` 快照。

### V2.0C：任务级仿真框架（Mission First）

状态：预览接入中，已完成最小任务级仿真闭环，插件市场继续作为后续任务模板和算法扩展基础设施。

目标是把平台从“单次场景求解”推进到“任务级仿真”：

- 任务场景：目标运动、观测、测向、波束控制、区域控场、风险评估和任务成功率代理；
- 任务时间线：离散时间步、对象状态更新、求解任务队列和结果回放；
- 任务模板：巡航目标、固定保护区、遮挡/反射环境、阵列退化和多目标任务；
- 任务指标：归一化场覆盖、保护区约束、鲁棒性、计算耗时和不确定度；
- Plugin Marketplace 继续作为传播后端、感知算法、防护/控场算法、报告模板和任务模板的基础设施。

当前预览已经完成：

- `hpm_platform.ui.mission_sim` 将旧版动态时间线求解器接入 V2.0 Scene First 工作台；
- FastAPI 暴露 `/api/mission/templates`、`/api/mission/status`、`/api/mission/run` 和 `/api/mission/results/{mission_id}`；
- 场景编辑首页新增“V2.0C 任务级仿真”面板，可选择任务模板、设置帧数、运行任务并查看指标/时间线/产物路径；
- 内置 `MST-TRACK-001`、`MST-DELAY-001` 和 `MST-STATIC-001` 三类任务模板；
- 每次任务运行输出 HTML 时间线报告、逐帧 CSV、压缩场数据、任务 JSON、ZIP 包和 `mission_v20c/index.json/csv`；
- 任务结果输出目标覆盖、跟踪误差、保护区 P95、归一化风险代理和任务成功代理，保留不输出真实毁伤概率、真实作用距离、现实作用距离、器件阈值或武器效能参数的安全边界；
- `plugins/builtin/` 内置传播后端、感知基准和报告模板三类 manifest；
- `hpm_platform.plugins.registry` 支持 manifest 解析、语义版本校验、参数 Schema、启停状态和验收摘要；
- FastAPI 暴露插件目录、详情、启停、运行和 V2.0C 验收清单 API；
- V2.0A 工作台新增本地插件市场页面；
- 插件运行限制在白名单 `builtin_hook`，不执行 manifest 指定的任意外部代码。

### V2.0D：论文自动生产线（Publication First）

状态：预览接入中，已完成 V&V 结果到论文草稿包的自动生成骨架。

目标是把 Publication Layer 工程化：

- 实验设计、批量扫参、重复试验和统计检验；
- 图表清单、论文图包、LaTeX/IEEE 模板；
- 摘要、方法、结果、局限性和附录的自动草稿；
- 表格编号、图题、引用占位和可复现实验索引；
- 一键生成论文草案与补充材料。

当前预览已经完成：

- `hpm_platform.publication.paper_factory` 可读取最新 V&V 机器结果；
- 自动生成 Markdown 论文草稿、IEEE LaTeX 骨架、图表清单 CSV、补充材料索引和论文包 ZIP；
- FastAPI 暴露 Paper Factory 状态、生成和下载端点；
- V2.0A 工作台“论文报告导出”页新增 Paper Factory 生成和下载控件；
- 论文包保留模型安全边界，不把归一化代理结果写成真实毁伤或作用距离结论。

### V2.1：数字孪生任务库

状态：路线图阶段。

目标是在 V2.0C 的任务级仿真框架上沉淀可复用任务模板、场景库和结果基准：

- 任务库：目标运动、阵列退化、反射环境、保护区约束、多目标控场和结果回放模板；
- 场景库：可发布的教学场景、论文复现场景和验收场景；
- 结果库：每个任务绑定输入场景、求解配置、归一化场分布、风险代理、V&V 复核和论文图表；
- 任务级对比：同一任务下的算法、传播后端、参数扰动和外部数据桥接对比。

## V3.0：真实数据接入与实验闭环

状态：预览接入中，已完成 CST/HFSS 导出包、测量数据批次、CSV、Touchstone、NPZ、HDF5 的导入审计层，以及外部数据 V&V 预评分风险审计。

目标是把 HPM-DT 从纯归一化公开模型扩展为可导入外部仿真和测量数据的科研平台。

规划能力：

- CST 导出包和工程元数据导入；
- HFSS/AEDT 导出包和工程元数据导入；
- 测量数据批次导入；
- CSV、Touchstone、HDF5、NPZ 数据解析；
- 统一数据集元数据、单位、坐标系、不确定度和标定准备度描述；
- 外部数据与平台降阶模型的误差对比、标定、测量不确定度传播和可信度评分。

当前预览已经完成：

- `hpm_platform.data_import.importers` 支持 CST/HFSS 导出包、测量批次、CSV、Touchstone、NPZ 和 HDF5 的格式识别；
- 内置样例覆盖近场 CSV、S2P Touchstone、复场 NPZ、HDF5 样例/签名桩、CST 导出包、HFSS/AEDT 导出包和 Measurement Campaign 测量批次；
- 导入器输出 SHA256、记录数、单位、坐标列、端口数、频点数、数组形状、工程元数据、仪器链、校准状态、不确定度模型、坐标规范化预览、标定准备度、预览记录和安全警告；
- FastAPI 暴露数据导入目录、验收、标定准备度、标定桥接、模型误差对比、样例解析和路径解析端点；
- V2.0A 工作台新增“数据导入”页面；
- 已把 Measurement Campaign 规范化复场样本桥接到 `CalibrationSamples`，并生成代理激励标定 smoke preview；
- 已生成导入样本驱动的模型误差对比预览，覆盖 RMSE、p95 残差、逐点残差和测量不确定度 1σ/2σ 覆盖率；
- 已生成外部数据 V&V 可信度审计，输出预评分、风险调整建议、正式纳入门槛和风险信号；当前因真实源链/相位参考未接入，不改写 V2.0A 核心评分；
- 当前阶段只做格式、元数据、校准、不确定度、标定准备度、接口桥接、代理模型误差审计和预评分传播，后续再进入真实源链/相位参考接入、标定闭环和正式 V&V 评分纳入。

## 当前约束

在 V2.x 阶段，平台仍保持归一化、公开、可复现、安全边界：

- 不输出真实毁伤概率；
- 不输出真实作用距离；
- 不输出真实器件阈值；
- 绝对功率只作为用户输入的实测标定元数据，不作为作用距离或阈值反推依据；
- 不冒充 CST/HFSS/COMSOL 全波仿真。

这些约束不是削弱平台定位，而是保证开源科研平台在算法验证、数字孪生方法和论文复现方向上保持可发布、可审计和可持续演进。
