# HPM-CAE Workbench V1.0 技术说明

## 1. 定位

V1.0 是一个配置驱动、本地运行、可复现的阵列算法 CAE。它不是全波电磁求解器，也不替代 CST/HFSS。当前物理层使用波长尺度几何、标量 Green 函数与归一化复场，主要用于阵列赋形、鲁棒性、动态控制和统计评价方法研究。

## 2. 软件分层

1. `project_model.py`：不可变数据类、几何校验、V0.9→V1.0迁移；
2. `quick_solver.py`：静态 Point-Focus、Region-LS、Nominal/Robust-PGMS；
3. `scene_editor.py`：浏览器 Canvas 拖拽交互，通过 Gradio 事件写回 Python；
4. `timeline.py`：目标轨迹、延迟观测、预测中心、逐帧赋形和代理评价；
5. `experiment_manager.py`：扫参定义、运行队列、95%CI、SQLite持久化；
6. `task_graph.py`：依赖闭包、拓扑排序、实时模块/适配器标识；
7. `app.py`：统一工作台与文件导出。

## 3. 动态时间轴

默认轨迹采用常速度/加速度项叠加横向正弦机动：

\[
\mathbf p_k = \mathbf p_0 + \mathbf v t_k + \frac{1}{2}\mathbf a t_k^2
+ [0, A\sin(2\pi t_k/T)]^T.
\]

控制器使用延迟观测时刻的中心位置，并可进行常速度外推。每帧采用快速 Nominal-PGMS 重新求解，再在**真实目标区域**而非设计区域上评价 RMSE、覆盖率和代理响应，从而避免“按预测区自评”的乐观偏差。

## 4. 批量试验数据库

数据库包含：

- `experiments`：项目名、参数范围、点数、重复次数、观察指标、状态；
- `runs`：参数值、重复序号、随机种子、状态、指标 JSON 和错误文本。

队列不会因单个无效几何点中断；失败项会作为正式记录保留。

## 5. 拖拽交互协议

Canvas 支持：

- 目标中心、长半轴、短半轴和旋转柄；
- 保护区中心与半径；
- 0.05λ 网格吸附；
- 松手后触发 `input` 事件，将几何参数写入 Python 组件，再调用统一项目校验和三维预览。

Python 数据模型始终是唯一持久化真源，前端画布不是第二套配置系统。

## 6. 当前边界

- 感知与接收防护在任务图中仍以 V0.3/V0.5 结果适配器形式接入；
- 当前动态任务聚焦空间控场，不把历史感知算法逐帧重跑；
- 归一化响应代理只用于算法排序，不对应器件毁伤概率；
- 浏览器端交互依赖现代 Chromium/Edge/Firefox 的 Canvas 与 ResizeObserver。

## 7. 下一版建议

V1.1 应把 V0.3 感知输出和 V0.5 防护权值作为正式任务节点接入统一运行上下文，并新增：节点级缓存、任务取消、并行 worker、可编辑多目标对象列表和可恢复会话。
