# Codex 接管说明

## 唯一主线

从本项目继续开发，不再向旧 `HPM_CAE` 的 `TargetRegion/ProtectedZone` 核心模型追加功能。

必须遵守：

1. 场景事实源是 `Entity + Components`；
2. 目标、接收器、阵列和环境均是通用实体；
3. 任务通过 `role:` / `component:` 查询选择对象；
4. 区域只属于探针、目标函数或约束，不是物理场景主角；
5. 新功能必须形成“建场景—建任务—运行—看结果—保存”闭环；
6. 首页不得重新变成评分、审计或论文报表后台；
7. 不输出真实毁伤距离、真实阈值或现实效能结论。

## 下一项开发任务

实现 0.2 Alpha 的 Three.js 三维编辑器，但不得改回固定数量对象逻辑。

最低验收：

- 使用 Three.js Scene/Object3D 维护与后端一致的场景层级；
- 用 TransformControls 编辑实体 Transform；
- 支持选择、移动、旋转、缩放、删除、复制、撤销和重做；
- 100个普通实体仍可交互；
- 所有变换通过 API 写回 `scene.json`；
- 示例工程保存并重开后位置一致；
- 保留现有17项测试并新增前端E2E测试。

## 禁止事项

- 不新增 `app_vXX.py`；
- 不新增“成熟度评分首页”；
- 不用对象名称判断算法行为；
- 不硬编码 `TGT-001`、`PRT-001`；
- 不为论文临时复制求解代码；
- 不提交大量运行结果和截图到源码主干。
