# HPM-CAE Workbench V1.1 技术说明

## 1. 定位

V1.1 是配置驱动、全 Python、本地运行的阵列算法 CAE。它的职责是把信号生成、阵列感知、接收端鲁棒波束形成、多目标归一化场控制、动态代理评价和实验管理放在同一个可复现工程中。当前物理层不是三维全波 Maxwell 求解器，而是阵列流形、相干多径统计模型、标量 Green 函数和无量纲概率/效用代理的组合。

## 2. 实时主链

```text
多对象场景
  ├─ 相干多径快拍 + 阵列幅相误差 + 局部坏通道
  │      └─ FBSS / PAWR / ESPRIT
  │             └─ DOA中心 + 谱宽不确定度
  │                    └─ MVDR / LCMV / Sector / CR-HybridNull
  └─ 目标区 + 保护区 + 功放/DPD
         └─ Point / Region-LS / PGMS / Robust-PGMS
                └─ 归一化场图与风险指标

感知质量 × 接收防护质量 × 控场质量 × 保护裕量
  └─ 归一化任务评分与全链路判据
```

`workflow_executor.py` 使用任务图自动补齐依赖。感知与接收防护不再是历史图片适配器，而是每次运行都从当前项目参数生成快拍并求解。

## 3. 多对象模型

V1.1 保留 V1.0 的主目标和主保护区字段，以兼容旧脚本，同时增加：

- `additional_targets`；
- `additional_protected_zones`；
- `interferers`；
- 每个对象唯一 `object_id`、名称和启用状态。

目标区采用旋转椭圆，保护区采用圆形。静态求解器将所有启用目标掩膜合并，并按 `amplitude_scale` 生成目标模板；保护区统计对所有启用区域求并集。交互模式限制为最多 6 个目标、6 个保护区和 4 个辐射源，以控制本地计算量。

## 4. 实时感知节点

默认场景包含一条直达路径和一条完全相干回波。数据生成包括：

- 8×8 阵列；
- 可配置快拍数和归一化 SNR；
- 阵元增益/相位随机误差；
- 可配置局部坏通道；
- 先验中心和先验宽度。

同一数据矩阵上并行评估：

1. FBSS-MUSIC；
2. PAWR-MUSIC；
3. FBSS-ESPRIT。

主输出包括二维空间谱、路径估计、球面角误差、协方差特征值、阵元可靠度和局部谱宽。局部谱宽被转换为接收端置信扇区，不再人为给接收算法一个完美角度。

## 5. 接收防护节点

候选方法：

- DL-MVDR；
- Point-LCMV；
- Sector-MVDR；
- CR-HybridNull。

CR-HybridNull 对每条已分辨相干路径建立置信扇区，联合软协方差加载和扇区主特征子空间硬约束。评价指标为归一化输出 SINR、真实路径最坏相对响应、白噪声增益和防护判据。所有功率均为相对量。

## 6. 任务队列

`job_queue.py` 使用 SQLite WAL 模式维护 `jobs` 与 `job_items`：

- 提交时展开扫参点和重复试验；
- 每个算例独立随机种子；
- 1—8 个 `ThreadPoolExecutor` worker；
- 算例完成即写入指标、耗时、worker 名称和错误；
- 暂停在当前并行批次结束后生效；
- 恢复只领取 `pending` 算例；
- 进程重启后残留 `running` 项自动回滚到 `pending`。

因此“暂停/恢复”不是前端按钮动画，而是数据库层真实检查点语义。

## 9. 研究诚信与边界

平台输出中的“效应”“风险”“可用率”是归一化代理变量。除非未来接入具有合法来源、适用条件明确并完成标定的数据，否则不能将代理评分解释为现实器件毁伤概率。报告会保留这一边界，避免用方向图或归一化场图替代链路预算和实测证据。
