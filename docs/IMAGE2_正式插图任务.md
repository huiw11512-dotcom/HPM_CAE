# Image2 正式插图任务

当前交付环境没有开放 image2，因此项目内没有冒充 image2 生成图。现有 `04_场景优先系统架构图.svg` 是可编辑矢量图，可直接用于技术文档。

后续在支持图像生成的环境中，按以下提示生成正式学术插图。

## 1. 场景优先系统架构图

生成一张16:9、白底、全中文、IEEE/Nature风格的高质量科研机理图。主题为“HPM-DT Studio 场景优先的系统级电磁任务仿真架构”。左侧展示两个阵列平台、三架沿不同轨迹运动的飞行对象、两栋建筑、固定接收设备和一个半透明场切片；中间依次展示“组件装配—任务定义—求解管线—结果工作区”；下方强调“对象不是目标1、任务不写死数量、结果视觉优先”；右侧以克制的橙色信息框标注“不替代CST/HFSS/COMSOL、不输出真实毁伤距离或器件阈值”。现代学术信息图，矢量质感，结构清晰，避免卡通和军事宣传风。

## 2. 城市多对象动态任务机理图

生成一张16:9中文科研插图。城市街区中有两个相控阵平台、三架沿不同轨迹运动的抽象飞行器、建筑物反射环境、一个固定接收器和动态场强切片；用细线表现阵列波束与运动轨迹，用色温表现归一化场分布。图中标注“物理场景、任务查询、动态求解、探针采样、时间线回放”。白底或浅灰底，工程软件说明图风格，不出现爆炸、武器毁伤或具体真实效能参数。

## 3. Entity + Components 领域模型图

生成一张全中文、可作为软件论文Figure的领域模型图。中心是通用“实体 Entity（UUID、名称、变换、标签）”，周围环绕几何、材料、阵列、发射、接收、运动、边界、探针、不确定度九类组件。右侧展示任务通过 `role:trackable`、`component:receiver` 等查询选择任意数量实体。突出“组合优于类型爆炸”“区域属于分析层”。颜色克制、扁平矢量、清晰箭头、无Graphviz默认外观。
