# HPM-CAE Workbench V0.9 技术说明

## 1. 目标

V0.9 的目标不是把已有图片塞进网页，而是建立一个稳定的“项目—求解—结果—导出”交互闭环，为后续逐步演进成电磁算法 CAE 平台打底。

```text
项目参数
  ↓ 校验与单位转换
场景几何
  ↓ 优化样本抽取
归一化场求解
  ↓ PA/DPD链路
全观察面评估
  ↓
交互图 + 指标 + 自包含报告 + 可复现结果包
```

## 2. 软件分层

### 表现层

`src/hpm_platform/ui/app.py`

使用 Gradio 构建本地浏览器界面，Plotly 提供三维与二维交互图。界面包含项目导航、参数检查器、视图区、求解监视器和结果工件区。

### 项目层

`src/hpm_platform/ui/project_model.py`

使用嵌套 dataclass 定义阵列、观察面、目标区、保护区和求解器参数。所有项目均可保存为 YAML，并在载入时重新校验几何边界和参数约束。

### 求解桥接层

`src/hpm_platform/ui/quick_solver.py`

将已有阵列几何、Green 函数、区域最小二乘、PGMS、场景鲁棒设计、PA和DPD封装为统一 `project -> result` 接口。小矩阵运算固定为单 BLAS 线程，避免桌面端多线程启动开销拖慢交互。

### 可视化与导出层

`src/hpm_platform/ui/figures.py`

输出三维场景、观察面场图、截线、远场方向图、阵元幅相和收敛曲线。

`src/hpm_platform/ui/exporter.py`

输出 YAML、CSV、JSON、NPZ、交互 HTML、SHA-256 清单和 ZIP。

## 3. 当前求解器

### Point-Focus

相位共轭单点聚焦。速度最快，作为场景检查和能量聚焦基线。

### Region-LS

以点聚焦相位作为模板，求解目标区复场拟合和区外惩罚的闭式最小二乘问题。

### Nominal-PGMS

对目标区幅度误差和区外铰链惩罚进行投影 Adam 优化，只使用名义场景。

### Robust-PGMS

在名义场景之外加入阵元增益、相位和目标配准扰动，最小化多场景平均幅度损失。

## 4. 默认工程

- 8×8 矩形阵，间距 0.5λ；
- 10 GHz，仅用于把 λ 映射为毫米；
- z=8λ 的 81×81 观察面；
- 旋转椭圆目标区；
- 独立圆形保护区；
- 5 场景 Robust-PGMS；
- 归一化 Rapp PA 与 DPD。

默认工程已经通过联合归一化判据，便于启动后直接观察一个有意义的结果，而不是空白界面。

## 5. 使用流程

1. 在左侧设置阵列、观察面、目标区和保护区。
2. 点击“刷新场景”只更新几何，不运行优化。
3. 选择求解器和不确定性参数。
4. 点击“运行求解”。
5. 在中部标签页检查场分布、截线、远场、阵元激励和收敛过程。
6. 在右侧下载项目、报告或完整结果包。

## 6. 当前边界

V0.9 仍是“阵列算法 CAE”，不是 CST/HFSS 的替代品：

- 场模型是自由空间标量 Green 函数近似；
- 尚无 CAD/网格导入、导体边界、介质、孔缝和腔体全波求解；
- 交互求解器目前针对固定二维观察面；
- 动态感知、防护和效应模块已在结果库中可浏览，但尚未全部改造成统一可编辑任务图；
- 求解任务是同步单作业，尚无后台队列和暂停/恢复。

这些边界会在后续版本中逐项拆解，不会用漂亮界面掩盖模型能力范围。
